At Kanaka, we recently built an AI platform for HR analytics. The interesting part wasn't the LLM integration itself, it was how we structured the agents to work together.

## The Problem

Most multi-agent setups I've seen use a fixed pipeline. Agent A runs, then Agent B, then Agent C. That works fine when your inputs are predictable. But in our case, a query could be asking for company reviews, market trends, talent benchmarking, or all three at once.

A fixed pipeline means you're always running agents that aren't needed, or worse, missing agents that are.

## The Agent Registry

Instead of hardcoding the flow, we built a registry. Each agent registers itself with a set of capabilities (basically keywords describing what it can do). When a query comes in, the orchestrator looks at what's being asked and dynamically picks the right combination of agents.

Here's a simplified version of what the registry looks like:

```python
class AgentRegistry:
    def __init__(self):
        self.agents = {}

    def register(self, name, agent, capabilities):
        self.agents[name] = {
            "agent": agent,
            "capabilities": capabilities
        }

    def get_agents_for_query(self, query):
        matched = []
        for name, entry in self.agents.items():
            if self._matches(entry["capabilities"], query):
                matched.append(entry["agent"])
        return matched
```

The key insight is that data sources are configurable per agent. If we need to add a new data source tomorrow, we register a new agent with the right capabilities and it automatically gets picked up for relevant queries.

## Context Management

The tricky part was maintaining context across dynamically sequenced agent calls. If Agent A pulls data from source X and Agent B needs that context to make sense of source Y, the orchestration layer needs to carry state forward.

We solved this with a shared context object that each agent reads from and writes to. The orchestrator passes it along sequentially.

## Results

The biggest win was the time savings. What took analysts 4-5 hours per company profile dropped to under 30 minutes. And adding new data sources went from a code change to a config change.

## What I Learned

Building this taught me that the orchestration layer matters more than the individual agents. A smart router with dumb agents outperforms a dumb router with smart agents every time.
